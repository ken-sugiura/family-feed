# ファミリーフィード 制作レポート

> 採用管理ワークスペース雛形をベースに、家族向け育児タイムラインアプリへ転用した記録。

---

## 1. ツールで解決できる課題

| 課題 | 従来の状況 | ツールによる解決 |
|------|-----------|----------------|
| 記録の分散 | 写真・メモが LINE / カメラロール / メモアプリにバラバラ | 1か所に集約してタグで整理 |
| 時系列の喪失 | いつ何があったか振り返れない | 月グループ付きタイムラインで一覧化 |
| 家族間共有 | 夫婦間でリアルタイムに共有する手段がなかった | Vercel 公開 URL を共有するだけでアクセス可能 |
| 成長記録の管理 | 身長・体重を別々のアプリで管理していた | イベント詳細に身長・体重欄を統合 |

## 2. 工夫したポイント

### 2-1. 複合フィルター設計
```
（人物 A OR 人物 B） AND （カテゴリ X OR カテゴリ Y）
```
- 「長男か次男」かつ「初めて・記念日」のイベントだけを一発で絞れる
- 従来の「1つだけ選択」では実現できなかった交差検索

### 2-2. 3秒投稿 UX
- タイトル（キャプション）1行を入力するだけで即投稿
- 詳細（写真 URL・メモ・身長・体重）は後から Pane 3 で編集可能
- 育児中の「両手が塞がる」状況を想定したスピード感

### 2-3. セージグリーン配色
- CSS トークン（OKLCH）を1か所変えるだけでアプリ全体の色が変わる設計
- 育児記録に合う「落ち着いた・やわらかい」トーンを選択
- ライト / ダークモードをヘッダーのボタン1つで切り替え可能

### 2-4. ドラッグ&ドロップ並び替え
- `@dnd-kit/sortable` を使いサイドバーの人物・カテゴリを直感操作で並び替え
- よく使う人物・カテゴリを上に置けるカスタマイズ性

### 2-5. ブランチ開発フロー
- 実験前に `git checkout -b feature/xxx` でフォーク
- 失敗しても `main` は安全な状態を保持
- OKならマージ → GitHub push → Vercel が自動再デプロイ

## 3. 苦戦したポイント

### フィルター中に追加したイベントが消えるバグ

**症状**
> 「ここに追加してもどこにも出てこない」

**原因**
- フィルターが有効な状態（例: 長男を選択中）で新規イベントを追加すると、
  そのイベントには `childIds: []` が設定される
- フィルター条件「長男が含まれる」を満たさないため即座に非表示になっていた

**解決策**
```ts
// 追加時にフィルター条件を自動付与
if (selectedPersonIds.length > 0) newEvent.childIds = selectedPersonIds;
if (selectedCategoryIds.length > 0) newEvent.categoryIds = selectedCategoryIds;
```
→ 追加直後から現在のタイムラインに表示されるようになった

## 4. その他気づいたこと

### 「フォークしてから実験」が安全網になる
